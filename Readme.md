# Resource Optimization Calculator

다양한 자원 간 변환 규칙을 기반으로 목표 아이템을 **최대한 많이 생산하기 위한 최적 전략**을 계산하는 웹 애플리케이션입니다.

> 단순 계산기를 넘어, 자원 변환 그래프와 탐색 알고리즘을 활용한 최적화 문제 해결 프로젝트입니다.

---

## 프로젝트 개요

여러 종류의 자원이 존재하고, 각 자원은 서로 다른 규칙을 통해 변환될 수 있습니다.  
이 프로젝트는 이러한 변환 관계를 기반으로 **제한된 자원 내에서 목표 생산량을 최대화하는 전략**을 계산합니다.

- 입력: 보유 자원 (여러 종류)
- 출력: 최대 제작 가능 수량 + 최적 교환 전략
- 추가: 부족 자원에 대한 보충 전략 (대체 경로 제시)

---

## ⚙️ 핵심 기능

### 1. 최대 생산량 계산
- 보유 자원과 변환 규칙을 기반으로 **최대 제작 가능 수량 계산**
- 단순 합산이 아닌 **변환 경로를 고려한 최적화**

---

### 2. 자원 변환 최적화
- 여러 변환 루트를 비교하여 **가장 효율적인 전략 선택**
- 불필요한 자원 낭비 최소화

---

### 3. 전략 분리 제공
두 가지 보충 전략을 분리하여 제공:

- 벌목의 가루 기반 전략
- 외부 구매(경매장) 기반 전략

👉 사용자가 상황에 맞게 선택 가능

---

### 4. 실시간 계산
- 입력값 변경 시 즉시 결과 반영
- UX 중심 설계

---

### 5. 상태 유지
- `localStorage`를 활용하여 입력값 유지

---

## 해결 방식 (핵심 알고리즘)

이 문제는 단순 계산이 아닌 **자원 최적화 문제**로 접근했습니다.

### ✔ 문제 특징
- 다수의 자원
- 비선형 변환 관계
- 목표 생산량 최대화

---

### ✔ 접근 방법

- **Brute Force 기반 탐색**
  - 가능한 제작 세트를 증가시키며 탐색

- **가지치기 (Pruning)**
  - 불가능한 경우 조기 제외

- **상태 기반 계산**
  - 각 단계에서 자원 상태를 계산하여 유효성 판단

- **탐색 범위 제한**
  - 성능 최적화를 위해 탐색 범위 제한 적용

---

### ✔ 설계 트레이드오프

| 항목 | 선택 |
|------|------|
| 정확도 vs 성능 | 제한된 탐색 범위 사용 |
| 완전탐색 vs 최적화 | 현실적인 시간 내 계산 가능하도록 조정 |
| 단일 전략 vs 다중 전략 | 사용자 선택 가능하도록 분리 |
