# Resource Optimization Calculator

다양한 자원 간 변환 규칙과 실시간 시세 데이터를 기반으로 목표 아이템을 **최대한 많이 생산하기 위한 최적 전략**을 계산하는 웹 애플리케이션입니다.

> 단순 수치 계산을 넘어, 자원 변환 규칙을 상태 탐색 방식으로 계산하고 경매장 시세를 반영해 제작/판매 손익까지 함께 판단하는 최적화 프로젝트입니다.

---

## 프로젝트 개요

Link: **https://abydos-app.vercel.app/**

여러 종류의 자원이 존재하고, 각 자원은 서로 다른 규칙을 통해 변환될 수 있습니다.  
이 프로젝트는 이러한 변환 관계를 기반으로 **제한된 자원 내에서 목표 생산량을 최대화하는 전략**을 계산합니다.

또한 Lost Ark 경매장 API를 연동하여 재료 및 완성품 시세를 실시간으로 불러오고,  
단순 제작 가능 수량뿐 아니라 **재료 판매 vs 제작 후 판매** 중 어떤 선택이 더 유리한지도 함께 보여줍니다.

- 입력: 보유 자원 (여러 종류)
- 출력: 최대 제작 가능 수량 + 최적 교환 전략 + 남은 자원
- 추가: 경매장 시세 기반 손익 비교 + 부족 자원 보충 전략

---

## 핵심 기능

### 1. 최대 생산량 계산
- 보유 자원과 변환 규칙을 기반으로 **최대 제작 가능 수량 계산**
- 단순 합산이 아닌 **자원 변환을 포함한 상태 탐색 기반 계산**
- 결과를 세트 단위와 최종 개수 단위로 동시에 제공

### 2. 자원 변환 최적화
- 여러 변환 루트를 비교하여 **가장 효율적인 교환 전략 선택**
- 불필요한 변환과 자원 낭비를 줄이도록 설계
- 자원 부족 시 어떤 자원을 먼저 변환해야 하는지 순서까지 안내

### 3. 경매장 API 기반 실시간 시세 반영
- 경매장 API를 통해 재료 및 완성품의 시세를 불러와 화면에 표시
- 별도 새로고침 없이 최신 시세 기준으로 손익 판단 가능
- 실시간 가격 변동을 반영하여 제작/판매 전략을 보다 현실적으로 제시

### 4. 전략 분리 제공
두 가지 보충 전략을 분리하여 제공:

- 벌목의 가루 기반 전략
- 외부 구매(경매장) 기반 전략

사용자는 상황에 맞는 전략을 선택해 참고할 수 있습니다.

### 5. 실시간 계산
- 입력값 변경 시 즉시 결과 반영
- 보유 자원 수량, 제작 수수료 감소율 등 입력에 따라 결과가 동적으로 변화
- 사용자 입력과 동시에 최대 제작 수량, 교환 순서, 손익 정보가 함께 갱신됨

### 6. 상태 유지
- `localStorage`를 활용하여 입력값 유지
- 새로고침 후에도 이전 입력 상태를 복원 가능

### 적용 목적

- 제작 후 판매하는 것이 유리한지 판단

## 해결 방식 (핵심 알고리즘)

이 문제는 단순 계산기가 아니라,  
**자원 변환 규칙을 포함한 상태 탐색 최적화 문제**로 접근했습니다.

### 문제 특징
- 다수의 자원이 존재
- 자원 간 변환 규칙이 존재
- 특정 자원은 다른 자원으로 교환 가능
- 목표는 제작 가능한 수량을 최대화하는 것
- 제작 이후 남는 자원까지 함께 고려해야 함

---

## 알고리즘 설계 방식

### 1. 목표 제작 수량 증가 탐색
알고리즘은 먼저 목표 제작 세트 수를 증가시키며,  
현재 보유 자원으로 해당 수량을 만들 수 있는지 반복적으로 확인합니다.

즉,

- 1세트 가능 여부 확인
- 2세트 가능 여부 확인
- 3세트 가능 여부 확인
- ...
- 가능한 최대 세트 수 탐색

이 과정을 통해 **최종적으로 제작 가능한 최대 수량**을 찾습니다.



---

### 2. 자원 변환을 포함한 가능 여부 판정
단순히 현재 보유 자원이 부족하다고 바로 실패하지 않고,  
다음 변환 규칙을 적용했을 때 제작 가능한지도 함께 계산합니다.

- 튼튼한 목재 5 → 목재 50
- 부드러운 목재 25 → 목재 50
- 목재 100 → 벌목의 가루 80
- 벌목의 가루 100 → 아비도스 목재 10
- 벌목의 가루 100 → 부드러운 목재 50

즉, **직접 보유량 + 변환 후 확보 가능한 자원량**까지 포함해 제작 가능 여부를 판단합니다.

---

### 3. 브루트포스 + 가지치기
각 목표 수량에 대해 가능한 변환 조합을 탐색하되,  
불가능한 경우는 빠르게 제외하도록 가지치기를 적용했습니다.

예를 들어:

- 필요한 아비도스 목재 수량이 절대 충족되지 않는 경우
- 변환 후에도 목재 수량이 부족한 경우
- 부드러운 목재가 최종 제작 조건을 만족하지 못하는 경우

이런 경우는 더 깊게 계산하지 않고 즉시 제외하여 불필요한 연산을 줄였습니다.

---

### 4. 남는 자원까지 포함한 결과 계산
최대 제작 가능 수량만 구하는 것이 아니라,  
제작 이후 남는 자원을 함께 계산하도록 설계했습니다.

남은 자원은 다음 정보로 분리해 보여줍니다.

- 남은 아비도스 목재
- 남은 부드러운 목재
- 남은 목재
- 남은 튼튼한 목재
- 남은 벌목의 가루

이렇게 남은 자원을 보여주는 이유는 다음과 같습니다.

- 다음 1세트를 만들기 위해 무엇이 더 필요한지 파악 가능
- 자원을 판매할지 보관할지 판단 가능
- 현재 전략이 얼마나 효율적으로 자원을 사용했는지 확인 가능

## 입력값에 따라 결과가 변화하는 계산 구조

이 프로젝트는 정적인 결과를 보여주는 것이 아니라,  
사용자가 입력한 값에 따라 결과가 실시간으로 달라지도록 설계했습니다.

입력값 예시:

- 아비도스 목재 보유량
- 부드러운 목재 보유량
- 목재 보유량
- 튼튼한 목재 보유량
- 제작 수수료 감소율

이 값들이 바뀌면 다음 결과가 즉시 다시 계산됩니다.

- 최대 제작 가능 수량
- 추천 교환 순서
- 부족 자원 보충 전략
- 교환 손익 체크
- 제작 vs 재료 판매 손익 판단
- 제작 후 남는 자원

즉, 단순 공식 대입이 아니라 **입력 상태 전체에 따라 결과 상태가 달라지는 구조**로 구현했습니다.

## 기술적 포인트

- **Brute Force 기반 탐색**
  - 목표 제작 세트를 증가시키며 가능한 최대 수량 탐색

- **Pruning**
  - 불가능한 조합을 조기에 제외하여 계산량 절감

- **상태 기반 계산**
  - 각 자원의 현재 상태와 변환 후 상태를 함께 고려

- **실시간 반응형 계산**
  - 입력값 변경 즉시 결과 갱신

- **시세 연동**
  - 경매장 API 가격을 반영한 손익 판단
